# Responsive Design

Verify the agreed route and state at a large-screen, tablet, and mobile viewport. Use the project's defined breakpoints when available; otherwise choose representative viewports and report the exact dimensions.

At each viewport:

- Verify that content reflows deliberately rather than merely shrinking.
- Verify that no text, controls, images, overlays, fixed elements, or floating controls overlap, clip, or obscure one another.
- Check for unintended horizontal scrolling, collapsed spacing, broken wrapping, and unreachable content.
- Preserve a usable reading order and interaction order as columns stack or controls move.
- Check relevant orientation and resize behavior when it can affect the layout.

Record the viewport and any exception in the verification report. Do not call the interface responsive until all required viewports have evidence.
