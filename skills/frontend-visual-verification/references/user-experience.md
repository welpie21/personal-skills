# User Experience

Identify the primary user goal, its starting point (A), and its completed outcome (B). Exercise that path in the rendered interface, including the states and validation a user encounters.

- Count the actions required to go from A to B. Identify actions that do not add required information, confirmation, safety, or user control, and remove or combine them when the task permits.
- Make the next intended action clear through labels, hierarchy, placement, and feedback. Do not rely on ambiguous icons, hidden expectations, or color alone.
- Provide one obvious path to the primary goal. Keep alternate paths only when they serve a clearly distinct user need, and label the distinction.
- Confirm that errors, loading, success, disabled, and empty states explain what happened and what the user can do next.

Report the tested flow, its action count, any removed steps, and any remaining ambiguity. Keep product decisions and required confirmations intact; reducing clicks must not reduce clarity, consent, or safety.
